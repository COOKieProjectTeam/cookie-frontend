# Защищённая ветка `main` и Pull Request

Ветка **`main`** на GitHub **защищена**.

## Обязательный поток изменений кода / конфигов

1. `git fetch origin` и создание рабочей ветки от **`origin/main`** (например `feat/issue-NN-kratko`, `fix/issue-NN-…`).
2. Коммиты только в этой ветке; затем **`git push -u origin <ветка>`**.
3. Оформить **Pull Request в `main`**; слияние после ревью и проверок.

**Не** пушить напрямую в `main`.

## Связка с задачей (issue)

- Форма задачи: `.github/ISSUE_TEMPLATE/task.yml`; **Trace** обязателен, при вертикали с API — **Companion** на issue в `cookie-backend`.
- В PR добавьте ссылку на issue (`Closes #NN` или `Refs #NN` по договорённости).
- Изменения без привязанной задачи — только если исключение согласовано с процессом команды.
